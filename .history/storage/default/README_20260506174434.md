# 🛡️ INDRA: EL MANIFIESTO AGNOSTICO (v1.0)
## Arquitectura de Soberanía e Inteligencia Modular

Este directorio `storage/default` es el **molde genérico** de la inteligencia de Indra. Todo sistema satélite debe heredar o sobreescribir esta estructura para garantizar la escalabilidad absoluta.

---

### 🏗️ 1. La Gran Separación: Seed vs. Satellite

*   **EL SEED (`src/`):** Es el **Motor Universal**. Contiene la fuerza bruta: React, Tailwind, el Bridge y los Proyectores (Blocks). El Seed es un actor mudo; sabe CÓMO dibujar, pero no sabe QUÉ está dibujando.
*   **EL SATELLITE (`storage/[tenant]/`):** Es la **Inteligencia Soberana**. Aquí reside el ADN del negocio, la estética de marca y las reglas lógicas. El Satélite es el director de orquesta.

---

### 🧬 2. Los 4 Pilares de un Satélite

Para crear un nuevo sistema desde cero (escalabilidad 100x), solo se deben configurar estos 4 elementos en la carpeta del cliente:

1.  **IDENTIDAD (`styles/tokens.css`):** 
    Define la "piel" del sistema mediante 20 variables CSS (`--sat-*`). No se toca el código de estilos del Seed; solo se inyectan los tokens de marca.
2.  **ADN (`db/schema_definitions.json`):** 
    Define los esquemas de datos y sus relaciones (Foreign Keys). Es el mapa relacional que permite al sistema "entender" la materia que procesa.
3.  **ENSAMBLAJE (`db/page_routes.json`):** 
    Define la estructura de las páginas mediante el stack de bloques agnósticos (Belt, Table, Form, Collection). Aquí se decide la jerarquía visual.
4.  **CEREBRO (`logic/`):** 
    Directorio de **Micro-Módulos Lógicos**. No es un monolito; cada archivo (ej: `pricing.js`, `stock.js`) es una inteligencia independiente que el Bridge del Seed escucha y ejecuta automáticamente.

---

### 🧠 3. El Orquestador de Eventos (The Hook)

La infraestructura del Seed incluye un **Router de Lógica** por default. Este componente:
1.  Escucha cada evento (Click, Write, Delete) que pasa por el Bridge.
2.  Escanea la carpeta `logic/` del Satélite activo.
3.  Distribuye el evento a todos los micro-módulos.
4.  **Soberanía Total:** El satélite puede interceptar, validar o modificar cualquier dato antes de que llegue a la base de datos, sin haber tocado una sola línea de código del motor central.

---

### 🚀 4. Protocolo de Escalabilidad (Clonación)

Para lanzar un nuevo proyecto en 30 minutos:
1.  **Clonar** la estructura de `storage/default` en `storage/[nuevo_cliente]`.
2.  **Inyectar** los colores de marca en `tokens.css`.
3.  **Definir** los campos de negocio en `schema_definitions.json`.
4.  **Arrojar** los scripts de cálculo en la carpeta `logic/`.

**Resultado:** Un ERP de grado industrial, visualmente único y lógicamente soberano, operando sobre el motor universal de Indra.
